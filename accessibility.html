Accessibility Statement

Commitment to Accessibility

I am committed to ensuring digital accessibility for people with disabilities. I am continually improving the user experience for everyone, and applying the relevant accessibility standards.

This website has been designed to conform to WCAG 2.1 Level AA accessibility standards. The website is built using semantic HTML5, ARIA landmarks, and keyboard navigation support to ensure all users, regardless of their abilities, can access and interact with the content.

Accessibility Features

List of Accessibility Features

  • Semantic HTML5: All pages use semantic HTML elements like <header>, <nav>, <main>, <article>, and <footer> for proper document structure.
  • ARIA Labels and Roles: Comprehensive ARIA implementation including aria-label, aria-labelledby, aria-describedby, and appropriate roles for enhanced screen reader support.
  • Keyboard Navigation: Full keyboard accessibility with visible focus indicators. All interactive elements are accessible via keyboard.
  • Skip Links: Navigation skip links to bypass repetitive content and jump directly to main content.
  • Color Contrast: Text and interactive elements meet WCAG AA color contrast ratios (minimum 4.5:1 for normal text).
  • Responsive Design: Mobile-friendly layout that adapts to all screen sizes and orientations.
  • Form Accessibility: All form fields have associated labels, clear error messages, and helpful hints.
  • Image Descriptions: Meaningful alt text for all images to provide context for screen reader users.
  • Readable Typography: Sufficient font sizes, line height, and letter spacing for readability.
  • Screen Reader Optimized: Content structure and live regions for dynamic content updates.

WCAG 2.1 Conformance

This website conforms to WCAG 2.1 level AA. WCAG stands for Web Content Accessibility Guidelines and is the international standard for web accessibility. It is developed through the W3C (World Wide Web Consortium) process in cooperation with individuals and organizations around the world.

The guidelines are organized around four principles often called POUR:

Perceivable
Information and user interface components must be presentable to users in ways they can perceive.
Operable
User interface components and navigation must be operable through various input methods including keyboard navigation.
Understandable
Information and user interface operations must be understandable to all users.
Robust
Content must be robust enough that it can be interpreted reliably by a wide variety of user agents, including assistive technologies.
